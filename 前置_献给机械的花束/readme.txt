操作台 (Console) 是机器人导入 NBT 方块时所使用的“控制台区域”，
它目前被定义为以操作台中心为中心的 11*5*11 的全空气区域。

您有责任确保以下事项。
1. 这个区域内都是空气
2. 这个区域内没有生物和其他方块的干扰
3. 机器人不会被命令方块或管理员移动
4. 机器人的背包不会因为 replaceitem/clear/give 等命令受到干扰
5. 操作台的位置远离 (0,0,0) 至少 30 格
6. 确保机器人不会受到其他未列出在以上的因素干扰

如果可能，我们建议你在导入期间关闭命令方块，因为任何干扰都可能造成严重问题。
另外，NBT 导入是交给另外一个机器人进行的，您需要自购相应的验证服务。

为了防止占用服务器空间，第二个机器人将不再会在一开始主动进入服务器。
请确保您在导入前已在控制台输入 “ffm”，然后我们将下载相应配套工具，并使第二个机器人进入服务器。
如果机器人在进服后没有管理员权限，则会向您索要。现在确保该过程只发生一次。

由于第二个机器人在一个独立的程序上运行，因此我们通过架设 HTTP 服务器进行信息传递。
因此，在第二个机器人进服并得到权限后，HTTP 服务器将会启动。
而后续如果导入 NBT 方块出现了问题，任何带“服务器”的文字都指代的是这个 HTTP 服务器而非租赁服。

另外，您只能通过 reload 或者 exit 使已启动的 HTTP 服务器关闭。

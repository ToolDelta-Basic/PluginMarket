# QQLinker v1.7.0 — 模块约定说明

模块开发遵循「约定优于配置」。框架通过模块类的属性声明自动完成服务注入/配置注册/事件订阅/命令注册/工具注册，模块只需声明，无需手动调用注册 API。

---

## 约定属性

| 属性 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `name` | str | ✅ | 模块唯一名称，用于日志/注册表/群过滤 |
| `version` | tuple | ✅ | 版本号，如 (1, 0, 0) |
| `mid` | int | — | 权限层级：0=kernel, 100=daemon, 200=service, 300=app。默认 300 |
| `tier` | int | — | mid 别名（deprecated，自动合并到 mid） |
| `dependencies` | list[str] | — | 依赖的模块名，按拓扑排序加载 |
| `required_services` | list[str] | — | 必需的服务名，缺失则跳过加载 |
| `background` | bool | — | 后台模块：即使在 ToolDelta 环境下也独立加载 |
| `enabled` | bool | — | False 则跳过加载。默认 True |
| `default_cooldown` | float | — | 命令默认冷却（秒）。默认 0 |

## 声明式 API

### @command 装饰器
```python
from qqlinker_framework.core.kernel.decorators import command

class MyModule(Module):
    @command(trigger=".ping", description="回复 pong", cooldown=3.0)
    async def cmd_ping(self, ctx):
        await ctx.reply("pong")
```

### @listen 装饰器
```python
from qqlinker_framework.core.kernel.decorators import listen
from qqlinker_framework.core.kernel.events import GroupMessageEvent, PlayerJoinEvent

class MyModule(Module):
    @listen(GroupMessageEvent)
    async def on_group_msg(self, event):
        if "你好" in event.message:
            await self.send_group_msg(event.group_id, "你好！")

    @listen(PlayerJoinEvent, priority=5)
    async def on_join(self, event):
        # priority=5 在 lane 内插队
        ...
```

### @tool 装饰器
```python
from qqlinker_framework.core.kernel.decorators import tool

class MyModule(Module):
    @tool(name="weather", description="查询天气")
    async def tool_weather(self, city: str) -> dict:
        return {"city": city, "temp": 25}
```

## 框架注入属性

模块实例化后框架自动注入：

| 属性 | 类型 | 说明 |
|---|---|---|
| `self.logger` | Logger | 模块专用 logger |
| `self.data_dir` | str | 模块数据目录（自动创建） |
| `self.db` | JsonDatabase | JSON 数据库代理 |
| `self.config` | ConfigProxy | 配置代理 |
| `self.group_config` | GroupConfigProxy | 群配置代理 |
| `self.game` | GameProxy | 游戏操作代理 |
| `self.qq` | QQProxy | QQ 操作代理 |
| `self.services` | ScopedView | 受限服务容器 |
| `self.event_bus` | LaneRouter | 事件路由器 |

## 配置文件

每个模块可在 L4 配置目录中拥有独立配置节：
```json
// 配置/模块/my_module.json
{
  "enable_feature_x": true,
  "max_retry": 3
}
```

通过 `self.config.get("enable_feature_x")` 读取。

## 事件订阅规则

### Lane 隔离
事件在诞生时即声明 lane，LaneRouter 按 lane 隔离执行：
- `critical` — SystemPanicEvent, SystemStartEvent, SystemStopEvent
- `admin` — ConfigReloadEvent
- `realtime` — GameChatEvent, PlayerJoinEvent, PlayerLeaveEvent, PlayerPositionEvent
- `chat` — GroupMessageEvent, PrivateMessageEvent, OutboundMessageEvent
- `ai` — AIResponseEvent, AIPrePromptReflectionEvent, AIPostResponseReflectionEvent
- `background` — 审计/日志

AI 处理阻塞 60s 不影响游戏事件 —— 不同 lane 隔离。

### 沙箱
非 root 模块（mid > 0）只能订阅白名单事件：
- GroupMessageEvent, PrivateMessageEvent
- PlayerJoinEvent, PlayerLeaveEvent
- GameChatEvent, ConfigReloadEvent

## Pipeline 路由

模块命令可通过 `pipeline_topic` 指定执行分支：
```python
@command(trigger=".封禁", pipeline_topic="admin_commands")
# admin_commands 分支：高优先、短超时、独立 worker

@command(trigger=".AI", pipeline_topic="ai_processing")
# ai_processing 分支：120s 超时、限并发、不阻塞其他命令
```

内置分支：`admin_commands` / `game_commands` / `ai_processing` / `general` / `background`

## 生命周期

```
Module.__init__()
  → _inject_magic_attrs()     注入 self.logger/self.db 等
  → _resolve_bridge()         解析 gatekeeper
  → _setup_config_schema()    注册配置

await module.on_init()        模块初始化（子类实现）

Module._post_init_conventions
  → 收集 @command / @listen / @tool 装饰器
  → 注册到 GatekeeperProxy
  → 订阅到 LaneRouter
  → 注入 config Proxy
  → 注册静态服务
  → 启动定时任务

await module.on_start()       模块启动（子类实现）

await module.on_stop()        模块停止
await module.on_freeze()      冻结
await module.on_thaw()        解冻
```

## 线程安全

- `ServiceRegistry` 使用 `threading.Lock`
- `LaneRouter` 内每条 lane 独立 asyncio.TaskGroup
- `PriorityEventQueue` 使用 `asyncio.Lock`
- 模块不应直接创建线程 —— 使用 `@scheduled` 定时任务
